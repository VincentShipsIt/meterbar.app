### Decisions

- **Decision:** Treat expired local OAuth credentials as disconnected.
  - **Context:** Local Codex and Claude Code tokens can exist while usage endpoints return 401.
  - **Rationale:** A disconnected state with a re-login prompt is more reliable than repeatedly making doomed usage calls.

- **Decision:** Keep Claude Code Keychain read for now.
  - **Context:** Local Claude Code tokens were only present in Keychain; `~/.claude/` had metadata but no access token.
  - **Rationale:** A file-only replacement would not work with the current CLI credential layout. The normal path now uses `claude /usage`; the Keychain OAuth reader remains only as an opt-in legacy fallback.

- **Decision:** Model extra Claude accounts as separate CLI config directories.
  - **Context:** CodexBar notes that Claude CLI usage is ambient to the active local CLI profile and can be mislabeled if treated as normal token accounts.
  - **Rationale:** Running `claude /usage` with a per-account `CLAUDE_CONFIG_DIR` gives each added account its own real credential/profile boundary without scraping or copying Claude-owned tokens.
