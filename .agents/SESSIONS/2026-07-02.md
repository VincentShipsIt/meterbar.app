## Session 1: DRY / slop audit (read-only, report deliverable)

**Status:** Complete
**Deliverable:** [docs/audits/01-dry-slop-audit.md](../../docs/audits/01-dry-slop-audit.md)

### What was done

Read every Swift/TS/YAML source file in the repo in full (app, widget, CLI, tests,
scripts, workflows) and reference-grepped all dead-code candidates. No code was
modified. A multi-agent audit workflow was attempted first but all subagents hit
the session usage limit, so the audit ran inline in the main context instead —
same evidence standard (direct reads + greps), no delegation.

### Key findings (detail + citations in the report)

- **Model layer forked 5×** (app / widget / CLI / 2 scripts) over one app-group
  JSON wire contract; drift already shipped one bug (CLI Int-vs-Double decode)
  and one live divergence (widget `iconName` semantics, k-only formatter).
- **4 incompatible severity threshold schemes** (UI ≤10/≤25 left; models ≥80/≥100
  used; notifications ≥90/≥100; CLI ≥50/≥80) + 3 `percentLeft` implementations
  with different rounding.
- **Popover vs dashboard are parallel implementations** — snapshot types, band
  labels, and status copy duplicated with drifted details (inverted Sonnet/Code
  Review label rule; hero icon derived by string-matching its own title).
- **`meterbar cost` ≠ app Costs tab**: CLI cost scan has no dedup, one root,
  file-mtime-only cutoff, hardcoded Sonnet pricing.
- **CI swallows test failures** (`test || echo` in ci.yml); coverage gate script
  exists but is not wired into CI.
- 17 dead/obsolete candidates confirmed by grep (unused symbols, test-only life
  support, identical `docs/icon.svg`, obsolete `create_xcode_project.sh`, dead
  CLI UserDefaults fallback, orphaned `.claude`/`.openai` admin providers,
  `.agent/` doc pointers → actual dir is `.agents/`).

### Recommendations shape

10-item ROI-ordered roadmap; safe mechanical deletes + CI hardening first, then
QuotaBands → ProviderSnapshot → shared MeterBarKit target. Explicit
non-recommendations included (no SQLite abstraction at 2 call sites; delete
trivial wrappers instead of generalizing).

### Next steps

- Review the report; roadmap items 1-2 (deletes + CI `|| echo` fix) are a safe
  first PR.
- CostTracker, CLI, widget, and notification logic have zero tests — add fixtures
  before any semantic refactor there (report §7).
